## 1. 앱 UI 구성 요약

현재 사용자 화면의 핵심 요소:

| 요소 | 설명 | 접근성 지원 |
|------|------|------------|
| **인사 배너** | "좋은 아침이에요" + 날짜시간 표시, 그라디언트 배경 | 텍스트만 존재, TTS 자동재생 없음 |
| **활동 헤더** | 이모지 + 활동명 + 과제명, 컬러코딩 배경 | Semantics 없음 |
| **현재 활동 요약 듣기** | OutlinedButton, 볼륨 아이콘 | TTS 수동 트리거 |
| **요리 뷰** | 가이드스크립트(StepsList) → 메뉴 선택(ChoiceChip) → 레시피 상세(준비물/재료/단계) | 각 단계별 TTS 버튼, 전체 듣기 버튼 |
| **운동 뷰** | 운동 방식 선택(앉아서/서서) → 단계별 안내 | 각 단계별 TTS 버튼 |
| **일반 뷰** | guideScript가 있으면 StepsList, 없으면 단순 텍스트 | TTS 버튼 |
| **사이드바 타임라인** | 600px 이상에서만 표시, 다음 활동 + 전체 타임라인 | 텍스트 기반 |
| **StepCard** | 번호 원형 배지 + 텍스트 + 볼륨 아이콘 버튼 | Semantics label 있음, 개별 TTS |
| **TTS** | 한국어, 속도 0.45, 볼륨 1.0 | 느린 속도로 설정됨 (좋음) |

**핵심 한계:**
- 이미지/사진/영상 콘텐츠 없음 (텍스트와 이모지에만 의존)
- TTS 자동재생 없음 (모든 TTS는 사용자가 버튼을 눌러야 작동)
- 활동 전환 알림/진동/소리 없음 (30초마다 화면 갱신만 수행)
- 완료 체크/진행 상태 추적 없음
- 도움 요청 기능 없음

## 2. 시뮬레이션 시나리오 설정

### 가정하는 일정 구성

```
08:00 - MORNING_BRIEFING: "아침 인사"
  guide_script: ["일어나서 세수해요", "옷을 갈아입어요", "아침 인사를 해요"]

12:00 - COOKING: "점심 만들기"
  guide_script: ["손을 깨끗이 씻어요", "앞치마를 입어요"]
  menus: [라면]
  (라면 레시피: 5단계 - 물 넣기→끓이면 면/스프 넣기→4분 대기→달걀 선택→불 끄고 옮기기)

15:00 - HEALTH: "운동하기"
  guide_script: ["편한 옷으로 갈아입어요"]
  (앉아서 운동: 4단계 / 서서 운동: 4단계)
```

## 3. 경증 (Mild DD) - 4명 시뮬레이션

### U1: 22세 남성, 호기심 많고 활동적

#### 시나리오 1: 오전 루틴 (08:00 아침인사)

**관찰 행동:**
- 앱을 열면 "좋은 아침이에요" 배너를 보고 미소를 지음
- "오늘도 하루메이트랑 함께 해볼까요?"를 읽고 "네!" 하고 대답함
- '현재 활동 요약 듣기' 버튼을 호기심에 눌러봄 → TTS가 "아침 준비 시간이에요. 오늘 할 일은 아침 인사 입니다." 읽어줌 → 좋아함
- 가이드 스크립트 단계를 순서대로 읽으며 따라함
- 각 단계의 볼륨 아이콘도 눌러봄 → TTS 재생됨 → 재미있어서 여러 번 누름

**성공률:** 95%
**앱이 도움 되는 부분:** 단계별 안내가 명확하고, TTS가 재미 요소로 작용하여 동기 부여
**앱이 실패하는 부분:** TTS를 장난감처럼 반복 재생하다 실제 활동을 안 할 수 있음. 완료 체크가 없어서 자기 진행 상태 파악 불가
**작동하는 UI:** 이모지 + 컬러코딩 활동 헤더, StepCard의 번호 배지, TTS 버튼
**작동 안 하는 UI:** 날짜시간 표시(관심 없음), 사이드바 타임라인(무시함)

#### 시나리오 2: 점심 요리 (12:00 라면 만들기)

**관찰 행동:**
- 활동이 바뀌면 화면이 주황색 계열로 변함 → "오 요리다!" 하고 흥분
- '오늘 요리 안내' 가이드를 빠르게 읽음 (손 씻기, 앞치마)
- 메뉴 선택에서 '라면' ChoiceChip을 정확히 탭함 → 선택 시 주황색으로 변하는 피드백 인지
- 준비물(라면냄비, 가스레인지) Chip을 읽음
- 레시피 5단계를 순서대로 따라감
- 3단계 "4분 동안 기다려요"에서 → 타이머가 없어서 감을 잡기 어려움, 대충 기다림
- '전체 듣기' 버튼을 눌러서 먼저 전체 흐름을 파악한 후 시작

**성공률:** 90%
**앱이 도움 되는 부분:** 메뉴 선택 → 레시피 연결 구조 직관적, 준비물 목록이 Chip으로 시각적
**앱이 실패하는 부분:** 타이머 기능 없음(4분 대기 단계), 완료 표시 없음, 불 관련 안전 경고 없음
**작동하는 UI:** ChoiceChip 메뉴 선택, StepCard 번호 순서, 전체 듣기 버튼
**작동 안 하는 UI:** 이미지 없는 재료 목록(라면 봉지를 어떻게 생겼는지 모르는 경우)

#### 시나리오 3: 운동 시간 (15:00 운동하기)

**관찰 행동:**
- 초록색 운동 헤더를 보고 인지
- '앉아서 하는 운동'과 '서서 하는 운동' 두 버튼을 보고 잠깐 고민
- 활동적이라 '서서 하는 운동' 선택
- 4단계를 따라하며 "두 팔을 올렸다 내려요. 10번" → 실제로 따라함
- 마지막 "수고했어요!" TTS를 듣고 뿌듯해함

**성공률:** 95%
**앱이 도움 되는 부분:** 운동 선택지 제공이 자율성 부여, 간단한 단계
**앱이 실패하는 부분:** 운동 시범 영상/이미지 없음, 횟수 카운터 없음

#### 시나리오 4: 활동 전환

**관찰 행동:**
- 시간이 지나면서 화면이 자동 갱신되는 것을 인지함 (30초 타이머)
- 다음 활동이 사이드바에 보이면(태블릿 사용 시) "아 다음에 이거 하는구나" 이해
- 전환 자체는 부드러우나, 알림이 없어서 앱을 보고 있지 않으면 놓침

**성공률:** 75%
**앱이 실패하는 부분:** 활동 전환 시 알림음/진동 없음, 화면을 안 보면 전환을 인지 못함

### U2: 30세 여성, 독립적, 혼자 하는 것 선호

#### 시나리오 1: 오전 루틴 (08:00 아침인사)

**관찰 행동:**
- 앱을 조용히 열고 인사 배너를 읽음
- TTS 버튼은 누르지 않음 (혼자 읽는 것 선호)
- 가이드 스크립트 3단계를 읽고 순서대로 수행
- 단계를 다 했는데 '완료' 표시가 없어서 약간 찝찝함

**성공률:** 98%
**앱이 도움 되는 부분:** 텍스트 기반 안내가 독립적 수행에 적합
**앱이 실패하는 부분:** 완료/체크 기능 없음, 자기 효능감을 느끼기 어려움
**작동하는 UI:** 깔끔한 단계 카드 레이아웃, 읽기 쉬운 폰트 크기(17px)
**작동 안 하는 UI:** TTS 강제 아님이 오히려 좋음 (선택적이라 독립성 존중)

#### 시나리오 2: 점심 요리 (12:00 라면 만들기)

**관찰 행동:**
- 요리 가이드를 차분하게 읽음
- '라면' 선택 → 준비물 확인 → 재료 확인 → 레시피를 처음부터 끝까지 한 번 읽음
- 그 다음 1단계부터 실행. 각 단계를 완료하면 스크롤해서 다음 단계로 이동
- 모든 단계를 정확히 수행하지만 진행 추적이 안 되어 "내가 몇 단계까지 했지?" 한 번 혼동

**성공률:** 92%
**앱이 도움 되는 부분:** 준비물/재료/단계 분리 구조
**앱이 실패하는 부분:** 현재 진행 단계 하이라이트 없음, 단계 완료 체크 없음

#### 시나리오 3: 운동 시간 (15:00 운동하기)

**관찰 행동:**
- '앉아서 하는 운동' 선택 (조용히 하는 것 선호)
- 4단계를 차례로 읽고 따라함
- 혼자서 문제없이 수행

**성공률:** 98%

#### 시나리오 4: 활동 전환

**관찰 행동:**
- 타임라인 사이드바(태블릿)를 통해 다음 일정을 미리 파악
- 시간 확인하며 스스로 전환을 인지
- 다만 앱이 자동 전환되면서 이전 활동의 레시피가 사라지는 것에 불안감

**성공률:** 85%
**앱이 실패하는 부분:** 이전 활동 기록/다시보기 불가

### U3: 19세 남성, 쉽게 산만, 빈번한 단서 필요

#### 시나리오 1: 오전 루틴 (08:00 아침인사)

**관찰 행동:**
- 앱을 열고 인사 배너를 1초 보고 다른 앱으로 전환
- 다시 돌아와서 "아 뭐였지?" → 화면에 가이드가 보이니 다시 읽기 시작
- 1단계 읽고 세수하러 감 → 세수하다 다른 생각 → 5분 후 돌아와서 앱 확인
- 앱이 같은 화면을 유지하고 있어서 2단계 확인 가능 → 도움이 됨
- TTS 버튼을 누르면 다시 집중하게 됨

**성공률:** 70%
**앱이 도움 되는 부분:** 화면이 유지되므로 돌아왔을 때 이어서 볼 수 있음, TTS가 재집중 도구
**앱이 실패하는 부분:** 자동 TTS 알림이 없어 집중을 다시 끌어올 수단 부족, 단계 완료 추적 불가(어디까지 했는지 혼동), 리마인더/알림 없음
**작동하는 UI:** StepCard의 시각적 구분(번호 배지)
**작동 안 하는 UI:** 모든 단계가 동일한 스타일 → 현재 단계 파악 불가

#### 시나리오 2: 점심 요리 (12:00 라면 만들기)

**관찰 행동:**
- 요리 화면에서 메뉴 선택까지는 도달
- '라면' 선택 후 레시피가 나오는데 단계가 5개 → 스크롤하다 3단계에서 주의 분산
- "물이 보글보글 끓으면" → 기다리다 핸드폰 딴짓 → 물이 넘침
- 앱이 타이머도 알림도 주지 않아서 위험 상황 발생 가능
- 결국 보호자 개입 필요

**성공률:** 45%
**앱이 도움 되는 부분:** 단계가 있어서 큰 그림은 파악 가능
**앱이 실패하는 부분:** 타이머 없음, 단계별 알림 없음, 안전 경고 없음, 현재 단계 하이라이트 없음

#### 시나리오 3: 운동 시간 (15:00 운동하기)

**관찰 행동:**
- 운동 선택 화면에서 두 버튼을 번갈아 눌러봄 (내용이 바뀌는 게 재미있어서)
- 결국 '서서 하는 운동' 선택
- 1단계 읽고 시작하지만 2단계에서 "10번"을 세다 잊어버림
- TTS를 다시 눌러서 확인

**성공률:** 60%
**앱이 실패하는 부분:** 횟수 카운터 없음, 단계 전환 자동 안내 없음

#### 시나리오 4: 활동 전환

**관찰 행동:**
- 활동 전환을 인지하지 못함 → 이전 활동을 계속 진행하거나 딴짓
- 알림이 울려야 전환을 인지할 수 있는 유형

**성공률:** 30%
**앱이 실패하는 부분:** 알림음/진동 전혀 없음이 치명적

### U4: 25세 여성, 완벽주의, 느리지만 꼼꼼

#### 시나리오 1: 오전 루틴 (08:00 아침인사)

**관찰 행동:**
- 인사 배너를 꼼꼼히 읽음, 날짜시간까지 확인
- '현재 활동 요약 듣기' 버튼을 눌러 TTS로도 확인
- 가이드 스크립트 각 단계를 읽고, TTS로 한 번 더 듣고, 수행
- 1단계를 완벽히 수행한 후 2단계로 넘어감 → 매우 느리지만 정확
- 완료 체크가 없어서 불안감: "진짜 다 한 건 맞나?"

**성공률:** 100% (시간은 2배 소요)
**앱이 도움 되는 부분:** TTS + 텍스트 이중 확인 가능, 단계별 분리 구조
**앱이 실패하는 부분:** 완료 확인/체크리스트 기능 없음 → 완벽주의자에게 불안 유발
**작동하는 UI:** 개별 단계 TTS 버튼 (확인 수단)
**작동 안 하는 UI:** 진행 상태 바 없음, 완료 피드백 없음

#### 시나리오 2: 점심 요리 (12:00 라면 만들기)

**관찰 행동:**
- 가이드를 먼저 전체 듣기로 한 번 듣고 시작
- 준비물 Chip을 하나하나 확인하며 실제로 꺼냄
- 재료도 하나하나 맞는지 확인
- 레시피 1단계: "물 550ml를 넣어요" → 계량컵으로 정확히 550ml 측정
- 3단계: "4분 동안 기다려요" → 핸드폰 시계 앱을 별도로 열어 타이머 설정 (앱에 없으므로)
- 5단계까지 완벽히 수행하지만 총 소요시간 30분 이상

**성공률:** 100% (시간 초과 문제)
**앱이 도움 되는 부분:** 상세한 단계 분리, 준비물/재료 명시
**앱이 실패하는 부분:** 내장 타이머 없음, 각 단계 예상 소요시간 없음, 완료 체크 없음

#### 시나리오 3: 운동 시간 (15:00 운동하기)

**관찰 행동:**
- 두 운동 옵션의 내용을 모두 확인한 후 선택
- 각 단계를 TTS로 듣고 정확히 따라함
- "어깨를 천천히 돌려요. 앞으로 5번, 뒤로 5번." → 정확히 5번씩 카운트

**성공률:** 100%

## 7. 핵심 발견 및 권고사항

### 7.1 앱의 강점 (유지해야 할 것)

1. **TTS 속도 0.45 설정**: 느린 말하기 속도가 인지에 도움. 모든 수준에 적합
2. **단계별 분리 구조 (StepCard)**: 번호 배지 + 개별 TTS가 순차 수행에 효과적
3. **컬러코딩**: 활동별 색상 구분이 직관적 (특히 중등도에서 색상 변화 인지에 유용)
4. **이모지**: 텍스트를 못 읽는 사용자에게도 부분적 단서 제공
5. **Semantics 적용**: StepCard와 ActivityCard에 스크린리더 지원 있음

### 7.2 시급한 개선사항 (우선순위)

| 우선순위 | 개선사항 | 대상 | 난이도 |
|---------|---------|------|--------|
| **P0** | 단계 완료 체크박스 + 진행률 | 경증/중등도 | 낮음 |
| **P0** | 활동 전환 알림 (소리/진동) | 전체 | 낮음 |
| **P1** | 이미지/사진 콘텐츠 추가 | 중등도/중증 | 중간 |
| **P1** | 자동 TTS 모드 (설정으로 on/off) | 중등도/중증 | 낮음 |
| **P1** | 요리 타이머 내장 | 경증/중등도 | 낮음 |
| **P2** | 안전 경고 시스템 (불/칼 단계) | 전체 | 중간 |
| **P2** | 격려/칭찬 피드백 시스템 | 전체 | 낮음 |
| **P2** | 도움 요청(SOS) 버튼 | 중등도/중증 | 중간 |

| **P3** | 감각 자극 인터페이스 | 중증 | 높음 |

### 7.3 결론

현재 HaruMate 사용자 화면은 **경증 발달장애인에게 적합하게 설계**되어 있으며, 텍스트를 읽을 수 있고 기본적인 스마트폰 조작이 가능한 사용자라면 높은 성공률(평균 80%)로 활용할 수 있습니다.

그러나 **중등도 사용자**(평균 40%)에게는 이미지와 자동 TTS가 없어 보호자 개입이 필수적이며, **중증 사용자**(평균 1%)에게는 사실상 사용 불가능합니다.

가장 적은 개발 비용으로 가장 큰 효과를 낼 수 있는 것은 **단계 완료 체크박스**와 **활동 전환 알림** 추가입니다. 이 두 기능만으로도 경증 사용자의 만족도가 크게 향상되고, 중등도 사용자에게도 진행 상태 파악이라는 핵심 기능을 제공할 수 있습니다.

중장기적으로는 **이미지 콘텐츠 파이프라인**을 구축하여 레시피, 운동, 일상생활 가이드에 사진/영상을 추가하는 것이 중등도 사용자의 독립 사용률을 실질적으로 높이는 핵심 전략입니다.
