# HaruMate(하루메이트) 코디네이터 화면 UX 워크스루

> 분석 기준 코드: `coordinator_screen.dart`, `home_screen.dart`, `api_service.dart`
> 작성일: 2026-03-23

---

## 앱 구조 요약 (분석 기반)

### 홈 화면 (HomeScreen)
- 상단 배너: "하루메이트 - 발달장애인을 위한 하루 도우미"
- 히어로 섹션: "오늘 하루, 같이 해봐요!"
- 기능 카드 2개:
  - **일정 만들기 (선생님용)** -> CoordinatorScreen으로 이동
  - **오늘 하루 보기** -> UserScreen으로 이동
- 사용 방법 안내 (3단계 스텝)
- 하단 안내 문구

### 코디네이터 화면 (CoordinatorScreen)
- AppBar: "코디네이터 - 일정 설계" + 뒤로가기 버튼
- **섹션 1:** 일정 내용 입력 (텍스트 필드, 기본값 미리 채워져 있음)
- **"일정 자동 만들기" 버튼** (AI 호출, 최대 30초)
- **섹션 2:** 자동으로 만들어진 일정 (ActivityCard 목록, 스와이프 삭제 가능, 탭하면 수정)
- **섹션 3:** 활동 자세히 설정하기 (확장형 카드, guide_script 표시, 수정 버튼)
- **섹션 4:** 오늘 일정 저장하기

### 수정 바텀시트 (_EditItemSheet)
- 시간(HH:MM), 할 일 텍스트 필드
- 안내 문장 (한 줄에 하나씩) 텍스트 필드
- **GPT로 수정 요청** 필드 (자연어로 AI에게 수정 지시)
- 저장 버튼

### API 서비스
- 일정 생성 (`/api/schedule/generate`): 텍스트 -> 일정 리스트
- 일정 수정 (`/api/schedule/edit`): 개별 항목 + 자연어 요청 -> 수정된 항목
- 서버 저장/로드, TTS, YouTube 검색, 이미지 검색
- 재시도 로직(최대 2회), 타임아웃 30초
- 한국어 친화적 에러 메시지

## 페르소나 1: 60대 엄마 (디지털 초보)

> **김순자** (62세), 스마트폰은 카카오톡과 전화만 사용. 발달장애 성인 아들(35세)의 일과를 관리해야 함. 앱 설치는 자녀가 해줌.

### 1. 단계별 행동

| 단계 | 행동 | 화면에서 보이는 것 |
|------|------|---------------------|
| 1 | 앱 아이콘 터치 | 홈 화면 로드. 상단에 "하루메이트" 배너, 이모지 손 흔들기 |
| 2 | 스크롤하며 화면 탐색 | "오늘 하루, 같이 해봐요!", 카드 2개, 사용 방법 안내 |
| 3 | "일정 만들기 (선생님용)" 버튼 발견 | **혼란**: "선생님용"이라고 되어 있어 본인이 눌러도 되는지 망설임 |
| 4 | 버튼 터치 | CoordinatorScreen 진입. "코디네이터 - 일정 설계" 타이틀 |
| 5 | 텍스트 입력 영역 확인 | 기본값이 이미 채워져 있음 ("08:00 - 오늘 일정 간단 안내" 등) |
| 6 | 기본값을 지우고 직접 입력 시도 | **혼란**: 어떤 형식으로 써야 하는지 모름. "08:00 - " 같은 포맷을 따라야 하나? |
| 7 | 대충 "아침 8시 밥먹기, 10시 산책" 입력 | 자유 형식으로 입력 |
| 8 | "일정 자동 만들기" 버튼 터치 | 로딩 인디케이터 + "일정 만드는 중... (최대 30초)" |
| 9 | 30초 대기 | **불안**: 화면이 멈춘 것 같아 다른 곳을 누르거나 앱을 끄고 싶어짐 |
| 10 | 결과 생성 완료 | 섹션 2에 ActivityCard 리스트 등장 |
| 11 | 카드를 보며 일정 확인 | 이모지와 시간, 할 일이 보임 |
| 12 | 섹션 3 "활동 자세히 설정하기" 발견 | 확장 화살표(expand_more)를 모름. 터치해야 하는지 인식 못함 |
| 13 | 스크롤 다운 → "일정 저장하기" 버튼 | 초록색 버튼이라 눈에 잘 들어옴 |
| 14 | "일정 저장하기" 터치 | "저장 완료!" 스낵바 |
| 15 | **혼란**: 저장 후 다음에 뭘 해야 하는지 모름 | 뒤로가기 → 홈 → "오늘 하루 보기"로 가야 하는데 그 흐름이 안내되지 않음 |

### 2. 혼란 포인트
- **"선생님용"이라는 라벨**: 부모도 사용할 수 있는 화면인데, "선생님용"이라고 하면 접근이 망설여짐
- **입력 형식 불명확**: 기본값은 "08:00 - 오늘 일정 간단 안내" 형태인데, 자유 형식도 되는지 안내가 없음
- **"코디네이터"라는 용어**: 60대 사용자에게 낯선 외래어. "일정 관리자" 또는 "보호자"가 더 직관적
- **AI 생성 대기 시간**: 30초는 디지털 초보에게 매우 길게 느껴짐. 프로그레스 바나 단계 표시가 없음
- **섹션 3 확장형 카드**: expand_more/expand_less 아이콘의 의미를 모름
- **저장 후 흐름 단절**: 저장 완료 후 "이제 '오늘 하루 보기'로 가세요"라는 안내가 없음
- **스와이프 삭제**: 스와이프로 항목 삭제가 가능한데, 이를 알 방법이 없음. 실수로 삭제할 위험

### 3. 만족 포인트
- 기본값이 미리 채워져 있어서 빈 화면보다 덜 당황함
- "일정 자동 만들기"가 직관적인 버튼 텍스트
- 초록색 저장 버튼이 눈에 잘 띔
- 에러 메시지가 쉬운 한국어로 되어 있음 ("와이파이를 확인해 주세요")
- 홈 화면의 사용 방법 안내 3단계가 도움됨

### 4. 개선 요청
1. "선생님용" -> "보호자/선생님용" 또는 아예 "일정 만들기"로 변경
2. 입력 필드에 "자유롭게 적어도 AI가 정리해 줍니다" 안내 추가
3. 저장 완료 후 "이제 '오늘 하루 보기'를 눌러주세요" 안내 팝업 또는 자동 이동 옵션
4. AI 생성 중 진행률 표시 또는 격려 메시지 ("열심히 만드는 중이에요!")
5. "코디네이터"를 "보호자/선생님" 같은 쉬운 말로 교체
6. 스와이프 삭제에 대한 온보딩 또는 삭제 버튼을 별도로 노출

### 5. 별점: 3.0 / 5.0
> 기본 기능은 작동하지만, 디지털 초보가 혼자서 완주하기엔 안내가 부족함. 특히 저장 후 흐름 단절이 큰 문제.

## 페르소나 2: 30대 특수교사 (매일 사용)

> **이민지** (33세), 특수학교 교사. 학급에 발달장애 학생 5명. 매일 아침 각 학생에 맞는 일정을 만들어야 함. 디지털 기기에 능숙.

### 1. 단계별 행동

| 단계 | 행동 | 화면에서 보이는 것 |
|------|------|---------------------|
| 1 | 앱 실행 → "일정 만들기" 즉시 터치 | 홈 화면은 빠르게 패스 |
| 2 | 학생 A 일정 입력 | 기본값 지우고 학생 A 맞춤 일정 타이핑 |
| 3 | "일정 자동 만들기" 터치 → 30초 대기 | 로딩 중 |
| 4 | 결과 확인 → 일부 항목 수정 필요 | 카드 터치 → 수정 바텀시트 |
| 5 | 수정 완료 → "일정 저장하기" | 저장 완료 |
| 6 | **문제 발생**: 학생 B 일정을 만들려면? | 뒤로가기 → 다시 "일정 만들기" 진입 |
| 7 | 학생 A의 입력값이 그대로 남아있음 | **혼란**: 이전 데이터가 초기화 안 됨. 수동으로 전부 지워야 함 |
| 8 | 학생 B 일정 입력 → 생성 → 저장 | 또 30초 대기 |
| 9 | 학생 C, D, E 반복 | **총 대기 시간: 약 2분 30초 (5명 x 30초)** |
| 10 | 5명 완료 | **문제**: 어떤 학생에게 어떤 일정을 저장했는지 구분 불가 |

### 2. 혼란 포인트
- **사용자(학생) 구분 기능 없음**: `saveScheduleToServer`에서 `'default_user'`로 하드코딩되어 있음. 학생별 프로필이 없음
- **이전 데이터 잔존**: 코디네이터 화면 재진입 시 `_textController`의 기본값이 하드코딩(생성자의 `text:`)되어 있어, 직전 작업 내용이 남아있을 수 있음
- **반복 작업 비효율**: 5명분을 만들려면 화면 진입 -> 입력 -> 생성(30초) -> 수정 -> 저장을 5번 반복
- **템플릿/복사 기능 없음**: 비슷한 일정 패턴(아침 루틴 등)을 재사용할 수 없음
- **저장된 일정 목록/히스토리 없음**: 이전에 만든 일정을 확인하거나 불러올 수 없음
- **날짜 선택 불가**: 오늘 날짜(`DateTime.now()`)로만 저장됨. 내일/다음주 일정을 미리 만들 수 없음

### 3. 만족 포인트
- GPT 수정 요청 기능이 강력함 ("시간을 19:30으로 바꿔줘" 같은 자연어 수정)
- AI가 guide_script까지 자동 생성해주어 안내 문장 작성 시간 절약
- 스와이프 삭제로 불필요한 항목 빠르게 제거 가능
- 수정 바텀시트에서 시간/할 일/안내 문장을 모두 편집 가능

### 4. 개선 요청
1. **사용자(학생) 프로필 관리**: 학생별 프로필을 만들고, 일정을 학생에게 할당하는 기능
2. **일정 템플릿**: 자주 쓰는 일정 패턴을 템플릿으로 저장/불러오기
3. **일괄 생성**: 여러 학생에게 동일 기본 일정 적용 후 개별 수정
4. **날짜 선택**: 달력에서 날짜를 선택하여 미래 일정 미리 만들기
5. **저장 히스토리**: 날짜별/학생별 저장된 일정 목록 조회
6. **빠른 초기화 버튼**: "새 일정 만들기" 버튼으로 입력 필드 빠르게 비우기
7. **병렬 생성**: 여러 일정을 동시에 생성 요청하여 대기 시간 단축

### 5. 별점: 2.5 / 5.0
> 1명분이라면 꽤 괜찮지만, 5명을 매일 관리하기엔 심각한 효율성 문제. 다중 사용자 지원과 템플릿이 반드시 필요.

## 페르소나 3: 25세 복지관 신입 사회복지사

> **박서연** (25세), 사회복지학과 졸업 후 장애인복지관에 첫 취직. "하루메이트 앱으로 이용자 일정 관리하세요"라는 지시만 받음. 매뉴얼 없음.

### 1. 단계별 행동

| 단계 | 행동 | 화면에서 보이는 것 |
|------|------|---------------------|
| 1 | 앱 실행 | 홈 화면. "하루메이트 - 발달장애인을 위한 하루 도우미" |
| 2 | 화면을 위에서 아래로 훑음 | 카드 2개가 보임: "일정 만들기", "오늘 하루" |
| 3 | "사용 방법 안내" 3단계를 읽음 | 1) 선생님이 일정 입력 저장 2) 오늘 하루 보기를 열어 켜둠 3) 지금 할 것만 크게 나옴 |
| 4 | "아 선생님=나구나" 이해 | **성공**: 사용 방법 안내가 전체 흐름을 잘 설명함 |
| 5 | "일정 만들기 (선생님용)" 터치 | CoordinatorScreen 진입 |
| 6 | 헤더 읽음 | "오늘 할 일을 입력하면 자동으로 일정표를 만들어 드려요" |
| 7 | 기본값 확인 | "08:00 - 오늘 일정 간단 안내" 등의 예시가 이미 있음 |
| 8 | **판단**: 이 예시를 수정해서 쓰면 되겠다 | 기본값을 부분 수정하여 이용자 맞춤 일정 작성 |
| 9 | "일정 자동 만들기" 터치 | 30초 대기 후 결과 생성 |
| 10 | 섹션 2 결과 확인 | ActivityCard들이 나열됨 |
| 11 | 카드 하나를 탭 | 수정 바텀시트 등장 |
| 12 | "GPT로 수정 요청" 필드 발견 | **혼란**: "GPT"가 무엇인지 모름. 일반 사용자에게 "GPT"는 기술 용어 |
| 13 | 일단 "안내 문장을 더 쉽게 바꿔줘"라고 입력 | 수정 결과 반영됨 |
| 14 | **만족**: "오 이게 AI인가? 신기하다" | AI 수정 기능에 감탄 |
| 15 | 저장 완료 | 스낵바 "저장 완료!" |
| 16 | 뒤로가기 → "오늘 하루 보기" | UserScreen으로 이동하여 결과 확인 |

### 2. 혼란 포인트
- **"GPT로 수정 요청"**: "GPT"라는 용어가 기술적. "AI에게 수정 부탁하기" 또는 "말로 수정하기"가 더 직관적
- **"guide_script"라는 라벨**: 섹션 3 확장 시 "안내 문장 (guide_script):"이라고 표시됨. 영어 기술 용어가 노출됨
- **"코디네이터"**: 앱 이름은 "하루메이트"인데 화면 제목은 "코디네이터". 용어 불일치
- **저장이 어디에 되는지 불명확**: 로컬? 서버? 다른 기기에서도 볼 수 있나? 안내 없음
- **활동 유형(type) 변경 불가**: 수정 바텀시트에서 type 필드가 `widget.item.type`으로 고정되어 있어 활동 유형을 바꿀 수 없음

### 3. 만족 포인트
- 홈 화면의 **사용 방법 안내 3단계**가 매우 유용함. 매뉴얼 없이도 전체 흐름 파악 가능
- 기본값이 채워져 있어 "이런 식으로 쓰면 되는구나" 이해 용이
- 헤더의 "오늘 할 일을 입력하면 자동으로 일정표를 만들어 드려요" 문구가 명확
- 에러 메시지가 친절하고 쉬운 한국어
- 전체 흐름이 1 -> 2 -> 3 -> 4 순번으로 되어 있어 순서 파악 쉬움

### 4. 개선 요청
1. "GPT로 수정 요청" -> "AI에게 수정 부탁하기" 또는 "말로 수정 요청"
2. "guide_script" -> "안내 문장" (영어 제거)
3. "코디네이터" -> "일정 관리" 또는 "선생님 모드"
4. 저장 위치에 대한 안내 추가 ("이 기기에 저장됩니다" 등)
5. 처음 진입 시 간단한 온보딩 투어 또는 툴팁
6. 활동 유형(type) 수정 기능 추가 (드롭다운 등)

### 5. 별점: 3.5 / 5.0

- **항목 추가 불가**: AI가 생성한 목록에서 삭제만 가능하고 새 항목 추가 UI가 없음

### 3. 만족 포인트
- **GPT 수정 요청이 매우 강력**: 자연어로 세밀한 수정 가능. "3단계로 나눠줘", "더 쉬운 말로 바꿔줘" 등
- 수정 바텀시트에서 시간/할 일/안내 문장 모두 편집 가능
- 시간 수정 시 자동으로 목록 재정렬됨
- 스와이프 삭제가 빠르고 직관적 (이 페르소나에게는)

### 4. 개선 요청
1. **항목 추가 버튼**: "+" 버튼으로 새 일정 항목 수동 추가
2. **활동 유형 변경**: 수정 바텀시트에 type 드롭다운 추가
3. **영상/이미지 첨부 UI**: `videoUrl`, `menus` 필드를 편집할 수 있는 UI 추가
4. **드래그 앤 드롭 재배치**: 직관적인 순서 변경
5. **미리보기 모드**: "사용자 화면에서 이렇게 보입니다" 프리뷰
6. **인라인 편집 옵션**: 바텀시트 외에도 목록에서 바로 수정 가능한 모드
7. **일정 복제**: 기존 항목을 복제하여 시간만 변경

### 5. 별점: 2.5 / 5.0
> AI 생성과 GPT 수정 기능은 인상적이지만, 수동 세밀 조정 기능이 부족. 파워유저에게는 편집 자유도가 낮음. 홈 화면에서 약속한 기능(영상 첨부)이 실제로 구현되지 않은 점은 신뢰도를 떨어뜨림.

## 페르소나 5: 40대 그룹홈 생활지원사

> **최영수** (45세), 발달장애인 그룹홈에서 생활지원사로 근무. 입소자 4명의 일과를 태블릿 1대로 관리. 각 입소자의 장애 유형과 수준이 다름.

### 1. 단계별 행동

| 단계 | 행동 | 화면에서 보이는 것 |
|------|------|---------------------|
| 1 | 태블릿에서 앱 실행 | 홈 화면 (400px 이상이면 카드 2개가 나란히 Row로 표시) |
| 2 | "일정 만들기" 터치 | CoordinatorScreen 진입 |
| 3 | 입소자 A(경증)의 일정 입력 → 생성 → 저장 | 정상 완료 |
| 4 | **문제**: 입소자 B의 일정을 만들려면? | 뒤로가기 → 다시 진입 |
| 5 | **문제**: 방금 저장한 입소자 A의 일정은 어디갔나? | `ScheduleStorage.save()`는 날짜 기반으로 하나만 저장하는 것으로 보임 |
| 6 | 입소자 B의 일정 입력 → 생성 → 저장 | **치명적 문제**: 입소자 A의 일정이 덮어쓰기 될 가능성 높음 |
| 7 | "오늘 하루 보기"에서 확인 | 마지막에 저장한 입소자의 일정만 보임 |
| 8 | **좌절**: 4명을 어떻게 관리하지? | 사용자 구분 기능이 전혀 없음 |
| 9 | 우회 시도: 기기 4대를 사용? | 비현실적. 예산과 관리 문제 |
| 10 | 우회 시도: 한 일정에 4명분을 모두 넣기? | 개인별 맞춤이 불가능해짐 |

### 2. 혼란 포인트
- **다중 사용자 관리 완전 불가**: 코드상 `'default_user'`로 하드코딩. 사용자 선택/전환 UI 없음
- **일정 덮어쓰기 위험**: `ScheduleStorage.save()`가 날짜 기준 단일 저장이면, 같은 날 2번 저장 시 이전 일정 소실
- **"오늘 하루 보기"가 단일 사용자용**: 한 화면에 한 사람의 일정만 표시
- **입소자별 커스터마이징 불가**: 장애 유형/수준에 따라 안내 문장 난이도를 다르게 해야 하지만, 이를 프로필로 관리할 수 없음
- **태블릿 최적화 미흡**: 코드상 `constraints.maxWidth < 400` 분기만 있고, 태블릿(800px+)용 레이아웃 최적화 없음

### 3. 만족 포인트
- 1명 기준으로는 일정 생성-수정-저장 흐름이 깔끔함
- AI가 guide_script를 자동 생성해주어 안내 문장 작성 부담 감소
- 태블릿에서 Row 레이아웃으로 카드가 나란히 표시되어 홈 화면은 보기 좋음
- 에러 발생 시 한국어 메시지가 친절

### 4. 개선 요청
1. **입소자(사용자) 프로필 시스템**: 이름, 사진, 장애 유형/수준 등록
2. **프로필별 일정 관리**: 프로필 선택 -> 일정 생성 -> 해당 프로필에 저장
3. **대시보드 뷰**: 4명의 오늘 일정을 한 화면에서 한눈에 확인
4. **프로필별 "오늘 하루 보기"**: 입소자 선택 후 해당 일정만 표시
5. **태블릿 레이아웃 최적화**: 큰 화면에서 사이드바+메인콘텐츠 2단 구조
6. **일정 복사**: A의 일정을 B에게 복사 후 수정하는 기능
7. **장애 수준 기반 AI 가이드**: "이 사용자는 경증이므로 간결하게" / "이 사용자는 중증이므로 단계를 세분화해줘" 옵션
8. **오프라인 지원 강화**: 그룹홈 와이파이가 불안정할 수 있으므로, 로컬 캐시 강화

### 5. 별점: 1.5 / 5.0
> 현재 구조는 완전히 1:1(코디네이터 1명 : 사용자 1명) 전용. 그룹홈처럼 1:N 관리가 필요한 환경에서는 사실상 사용 불가. 가장 시급한 개선 영역.
